Fix the Jarvis Hook Model. There are 3 critical bugs making it predict "1 view" for everything:

## BUG 1: Normalizer is empty
The model-v2.json has normalizer: {} — no mean/std for any feature.
This means every feature z-scores to garbage (division by zero defaults to raw value, then clips to ±5).

FIX: Compute actual normalizer stats from the 372 training videos.
For each pre-upload indicator (transcript_word_count, unique_word_ratio, etc.) at each time window,
compute the actual distribution across training videos and store mean + std.

The training data is at:
/Users/tylercsatari/Desktop/BusinessHub/tyler_ml_dataset/01_video_performance/videos_complete.json

Each video has:
- transcript_words: [{word, timestamp_s}] — word-level timestamps
- transcript_text: full transcript text
- total_views: actual view count
- avg_percent_viewed, swiped_away_rate_pct, stayed_to_watch_pct, avg_retention_vs_baseline, non_sub_fraction

To compute the normalizer:
1. For each of 372 videos, extract hook text at @1s, @3s, @5s, @10s (using word timestamps)
2. Compute each indicator value at each window:
   - transcript_word_count: len(words_in_window)
   - transcript_char_count: len(' '.join(words_in_window))
   - unique_word_ratio: len(set(words)) / max(len(words), 1)
   - hapax_legomena_ratio: words appearing once / total
   - hook_question_count: count of '?' in window
   - hook_question_density: question_count / max(word_count, 1)
   - exclamation_count: count of '!'
   - repeated_phrase_count: bigrams appearing 2+ times with gap >= 10 words
   - hook_word_ratio: len(words_in_window) / max(len(all_words), 1)
   - pivot_word_count: count of ['but','however','yet','although','whereas','while','nevertheless','meanwhile','despite','instead','rather','conversely','nonetheless','on the other hand','in contrast']
   - pivot_word_density: pivot_count / max(word_count, 1)
   - comparison_word_count: count of ['more','less','than','fewer','greater','larger','smaller','better','worse','faster','slower']
   - second_person_ratio: count of ['you','your',"you're","you'll","you've","you'd"] / max(word_count, 1)
3. For each feature, compute mean and std across all 372 videos
4. Save to model-v2.json normalizer field

## BUG 2: Post-node r-values are None
The 6 post-upload nodes have r_value: None, meaning they contribute 0 to the output.

FIX: Compute actual Pearson r between each post-upload metric and log10(views) from the 372 videos.

For each post metric (avg_percent_viewed, swiped_away_rate_pct, stayed_to_watch_pct,

Store these as r_value in the post_nodes.

## BUG 3: Post-upload nodes need all atomic indicators, not just 6

The post-upload layer should include ALL significant post-upload indicators, not just 6 basic fields.
From the retention curves in videos_complete.json, derive atomic indicators:
- ret_at_10pct: retention value at 10% of video

- ret_at_75pct: retention value at 75%

- max_cliff: most negative single-step retention drop
- deriv_std: standard deviation of per-step retention changes

For each, compute Pearson r with log10(views) across 372 videos.
Include all where |r| >= 0.05.
